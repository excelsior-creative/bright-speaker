## What I learned

- The sticker-brand design system is genuinely portable. The
  existing utility classes (`.btn`, `.pill`, `.card`, `.step`,
  `.faq-item`, `.feature-card`, `.eyebrow`, `.lede`) covered
  ~95% of what the six educator pages needed. Only two pages
  (contact, blog index) needed a tiny amount of bespoke styling
  for the hover-nudge on clickable cards.
- The `for-educators` audience-block rewrite exposed a real risk
  I hadn't named yet: every time we edit the audience copy, we
  risk re-asserting the old 6–12 ordering. Long-term fix: a
  single source-of-truth audiences array that gets sorted/filtered
  per page. Low priority, but worth it after the next content
  pass.
- Lint rule `react-hooks/set-state-in-effect` is statically
  conservative — it doesn't know `await` makes a setState
  asynchronous. Inlining the body (or adding a cancelled guard
  inside an IIFE) is the durable fix; suppressing the rule with
  a comment would have worked but would paper over the real
  cancellation-safety issue.

## Notes for future-me

- The `/speak/page.tsx` effect now has an IIFE pattern with a
  `cancelled` flag. If you need to add more async init work, do
  it inside the same IIFE and gate setState calls on `!cancelled`.
- If you need to add more educator pages, you can lean heavily on
  the patterns established tonight:
  - Hero: `<span className="pill mb-5">` + `<h1 className="text-[clamp(36px,4.2vw,58px)]">` + `<p className="lede">`.
  - Grid of features: `<div className="feature-card">` + 10×10
    rounded icon swatch with ink border.
  - CTA bands: `.band-ink` + rounded-[28px] + thick border +
    explicit sticker shadow.
- When removing compliance words from copy, also grep the file
  for `DPA`, `SOC 2`, `FERPA`, `COPPA`, `Student Privacy Pledge`,
  and `MediaPipe` so a single-word fix doesn't miss a
  sister-sentence. That was the bug pattern night 2 called out.
